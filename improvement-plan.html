Brick 改善計畫

目標：對齊 getbrick.com 的核心體驗——不碰實體 NFC key，就是解不開。

2026-07-09 · 基於 main branch code review

✅ 本輪已完成

DONE1. 移除 Release 版手動解鎖後門

首頁 Manual Stop 與 Settings 的 End Block Now 改為 #if DEBUG 限定。 正式版唯二解鎖路徑：掃已配對 NFC key（含 5 秒冷靜倒數確認）、或消耗一次 Emergency Unbrick。

ContentView.swift · Release build 驗證通過

DONE2. Blocking 中禁止配對新卡

原本 block 進行中掃任意陌生卡就能配對、再掃一次即可解鎖，等於免費後門。 現在 blocking 期間掃到未配對的卡直接拒絕，不進入配對流程。

BlockSessionModel.handleKeyScan · 新增單元測試 testUnknownKeyDuringBlockIsRejectedWithoutPairing

DONE3. 拒絕無 identifier 的 NFC tag

空 identifier bytes 全部 hash 成同一個 fingerprint，任兩個回空 ID 的裝置可互相解鎖（YubiKey over ISO 7816 常見）。 現在掃描階段直接拒收並顯示錯誤，不允許配對。

NFCFingerprint.scannedKey 改回傳 Optional，掃描器對 nil 直接 invalidate

🗺️ 功能 Roadmap

P1無限期模式（Brick 的靈魂）

getbrick 的主模式是「鎖到你回去碰那塊磚為止」，目前只有 15 分～8 小時倒數。 在 duration 選項加一個 ∞：無 endsAt、無 timer、只有 NFC 或 emergency 能解。

P1DeviceActivity 自動到期

目前 shield 到期清除只靠 app 內 Timer，app 被殺掉後會鎖到下次開 app 為止，跟 UI 顯示的剩餘時間不符。 用 DeviceActivityCenter 排 schedule + Monitor extension 在 intervalDidEnd 清 shield。

P1Emergency 次數搬進 Keychain

現在放 UserDefaults，刪 app 重裝就回滿 5 次。Keychain 在 reinstall 後仍保留，一行 wrapper 就能換。

P2NDEF 背景喚醒

iPhone 對 NDEF tag 有系統級背景讀取：tag 寫入 universal link，碰一下手機直接拉起 app 進入解鎖流程。 這才是「拿手機碰磚」的正確姿勢，也順便解掉每次進前景就彈 NFC sheet 的干擾。

P2移除裝飾性的 targetName

「Reddit」只是顯示文字，實際 block 的是 FamilyActivitySelection，兩者可以完全對不上。 砍掉欄位，從 selection 摘要自動生成顯示名稱。

P3防刪 app 提示 / 上架準備

Family Controls entitlement 正式發佈需要跟 Apple 申請；另外評估 Screen Time 限制 app 刪除的引導文案。

🎨 UI 改善計畫

#項目內容優先
U1Bricked 全螢幕鎖定畫面 blocking 時整頁純黑、隱藏 TabBar，只剩鎖頭、倒數大字、「Scan to Unbrick」。狀態一眼可辨，也呼應 getbrick 的黑白品牌感。 P1
U2首次啟動三步精靈 選 Apps → 掃卡配對 → 完成。目前新用戶要自己發現「Choose Apps」跟右上角小 NFC icon，流程散在兩個 tab。 P1
U3狀態訊息改 toast statusMessage 常駐面板像 log console。改成 3 秒自動消失的 toast，畫面留給主狀態。 P2
U4Duration 改 preset chips Stepper 按 31 次才能從 15m 到 8h。改成 30m / 1h / 2h / 4h / ∞ 一排 chips。 P2
U5Settings 收納 Diagnostics + Debug 併成一個 NavigationLink 子頁；Emergency 移到最底、加紅色框強調「這會消耗次數」。 P2
U6掃卡回饋 掃到卡成功 / 拒絕時加 haptics（success / error），鎖頭開合加 spring 動畫。廉價但體感差很多。 P3
